# Nextdocs — руководство пользователя

Nextdocs — это платформа для совместного написания документации. Вы
создаёте проект (или подключаете GitHub-репозиторий), пишете страницы в
живом редакторе, приглашаете команду — и получаете полноценную базу
знаний с историей версий, поиском и AI-агентом, который умеет как
отвечать на вопросы, так и редактировать документацию по просьбе.

Эта папка — **руководство для конечного пользователя**. Каждый файл
описывает одну фичу: зачем она, где её найти и как пользоваться.

## Философия: всё из коробки, без плагинов

Классический подход конкурентов — «голый wiki-движок + маркетплейс
плагинов». Нужны диаграммы? Купите Gliffy или Lucidchart. AI-поиск?
Подключите Rovo / Notion AI за отдельные деньги. Импорт PDF? Ищите
сторонний addon. Коллаборация в реальном времени? Это в Premium
плане. И собирайте пазл сами.

Nextdocs собран по другому принципу: **всё нужное — из коробки,
интегрировано, бесшовно**.

- **Редактор + коллаборация + живые курсоры + follow-mode** — не
  отдельные продукты, а одна система.
- **Mermaid + TLDraw + Excalidraw диаграммы** — встроены, с поиском
  по их тексту, с real-time collab.
- **AI-агент + полнотекстовый поиск + семантический поиск по
  документации и коду одновременно** — единый интерфейс, ничего не
  надо подключать. Агент умеет не только искать и объяснять, но и
  создавать/редактировать страницы по вашей команде (с обязательным
  подтверждением). Код репозиториев — только читает.
- **Импорт из Confluence / Notion / Google Docs / PDF / Excel /
  markdown / ZIP / GitHub** — каждое расширение парсера написано в
  основе, а не отдельным плагином.
- **Версии, комментарии, упоминания, уведомления, права, интеграция
  с GitHub** — основные фичи, а не опции.

Это важное решение с обеих сторон:

- **Плюс:** вы не тратите 2-3 недели на «обвязку»: регистрации
  плагинов, настройка SSO, прокидывание тарифов отдельно для AI,
  отдельно для диаграмм. Всё работает сразу.
- **Минус:** если у вас есть специфичная потребность («хотим
  интеграцию с нашим собственным HR-порталом»), прикручивать её
  через плагин-маркет нельзя — нужно просить фичу через нашу
  поддержку / roadmap. Мы работаем быстро, но это не «сегодня
  поставил плагин».

Для большинства инженерных команд это правильный выбор: меньше
времени на сборку, меньше SaaS-бёрна, меньше мест где что-то
отвалится.

## Глоссарий

- **Проект** — контейнер для связанных документов (например, один
  микросервис, один продукт, одна команда).
- **Страница** — одна markdown-документ внутри проекта. Имеет своё
  дерево вложенности, историю, комментарии.
- **Репозиторий** — связь с GitHub-репой. Один проект может быть
  привязан к нескольким репам одновременно.
- **Yjs / живое редактирование** — механика, по которой несколько
  человек могут одновременно печатать в одной странице без конфликтов.
- **RAG** — AI-поиск с семантикой. В отличие от обычного поиска
  работает не только по совпадению слов, но и по смыслу.
