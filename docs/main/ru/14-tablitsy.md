# Таблицы

## Зачем это нужно

Таблицы — один из самых частых способов структурировать данные в\
документации: сравнения решений, матрицы ответственности, план\
релизов, списки endpoint'ов, конфигурации. И одновременно — одна из\
самых раздражающих фич в большинстве wiki-инструментов: жёсткие\
размеры колонок, кривая вставка из Excel, невозможность редактировать\
нормально.

В Nextdocs таблица — полноценный первоклассный блок. Все операции\
с ней — через всплывающее меню, без модальных окон и настроек.\
Работать с ней нужно так же быстро и естественно, как в Excel или\
Google Sheets.

## Вставка таблицы

### Из слеш-меню

`/` → **Table** → вставляется таблица 3×3. Первая строка по умолчанию\
— header (жирный, фиксированный шрифт).

### Из буфера обмена

См. раздел «Вставка из буфера обмена» в [Editor](02-editor.md):

* **Из Excel / Google Sheets** — Cmd+C диапазона → Cmd+V в Nextdocs\
  → таблица с правильными размерами и содержимым.

* **CSV / TSV** — текст с запятыми / табуляцией превращается в\
  таблицу автоматически.

* **HTML table** — если копируете страницу с web'а, где есть `<table>`,\
  она становится нашей таблицей.

* **Markdown pipe-table** — стандартный markdown-синтаксис\
  (`| col | col |`) парсится.

## Плавающее меню над таблицей

Наведите курсор на таблицу — над её верхней кромкой появляется\
панель. Оттуда:

### Добавление строк и колонок

* **Add row** / **Add column** — добавляет строку ниже / справа от\
  текущей позиции курсора.

* Альтернатива — Tab в последней ячейке последней строки\
  автоматически создаёт новую строку.

* Сочетания клавиш: Cmd/Ctrl+Shift+↓ / → — добавляет строку / колонку.

### Удаление

* Выделите строку / колонку (клик по индикатору слева / сверху) →\
  кнопка «Delete row/column» в меню.

### Reorder

* Перетаскивание индикаторов строк / колонок мышью — меняет\
  порядок. Работает с анимацией, можно сразу увидеть куда\
  переедет.

### Ячейки

* Merge / split cells через контекстное меню (правая кнопка).

* Выравнивание (left / center / right) для текста в ячейке.

* Background-цвет ячейки для подсветки важного.

## Форматирование содержимого

В каждой ячейке работает полный набор форматирования:

* Жирный / курсив / inline-код / ссылка.

* Многострочный текст (Shift+Enter внутри ячейки — перенос строки).

* Чек-листы внутри ячейки (`/checklist`).

* Маркированные / нумерованные списки.

* Изображения — ячейка масштабируется под картинку.

* Ссылки на другие страницы (`@` + название страницы — тянется\
  ссылка).

## Полноэкранный режим

Кнопка «расширить» в левом нижнем углу таблицы — открывает её в\
отдельном окне поверх всего. В fullscreen:

* Видно всю таблицу целиком без горизонтального скролла.

* Удобно работать с таблицами на 20+ колонок.

* ESC / клик по «сжать» — возврат в редактор.

Результат автоматически синхронизируется: что изменили в\
fullscreen-режиме, сразу применяется в исходном редакторе (и у\
коллег).

## Экспорт в Excel

Кнопка **Save as Excel** (в меню над таблицей). Скачивается .xlsx\
файл с содержимым текущей таблицы. Поддерживаются:

* Содержимое ячеек (текст, числа).

* Заголовки строк.

* Порядок колонок.

Не переносится: форматирование (жирный / курсив), цвета, hyperlink'и\
(вставляется сам текст). Для пиксельной совместимости с Excel лучше\
делать таблицы прямо там.

## Копирование как HTML

Выделили таблицу в Nextdocs → Cmd+C → вставили в Gmail / Slack /\
Word. Таблица сохраняется как HTML-таблица, визуально близкая к\
тому что было. Удобно для рассылки отчётов.

## Сортировка и фильтры

Пока нет. Таблицы в Nextdocs — в первую очередь средство\
представления данных для людей, а не мини-спредшит. Если нужны\
pivot / sort / formulas — делайте в Google Sheets и вставляйте\
готовое в документацию.

## Совместное редактирование

Таблицы — такой же CRDT-документ, как весь остальной редактор:

* Два человека могут одновременно редактировать разные ячейки без\
  конфликтов.

* Курсор коллеги виден в ячейке, где он сейчас печатает.

* Если оба правят одну ячейку — характер слияния такой же, как у\
  обычного абзаца: символы interleavятся, потерять что-то\
  невозможно.

## Глубокое связывание

У каждой ячейки можно получить ссылку через правый клик → **Copy**\
**cell link**. Это `https://nextdocs.ai/projects/.../page#cell-ab12`,\
которую можно кинуть в Slack — открывающий попадёт прямо на ячейку,\
ячейка подсветится.

## Поиск

Полнотекстовый поиск (Cmd/Ctrl+K, фильтр **Table**) ищет по\
содержимому ячеек. Найденный результат подсвечивает конкретную\
ячейку при открытии страницы.

## Типовые сценарии

**Сравнение продуктов** (как таблица в [13-comparison.md](13-comparison.md)).\
Пишете список свойств по строкам, продукты по колонкам. Заливаете\
✅ / ❌ / ⚠️. Одна таблица заменяет 5 абзацев текста.

**Матрица ответственности (RACI)**. Роли по колонкам, активности по\
строкам. Ячейки — R / A / C / I. Разглашается в команде через\
ссылку на ячейку при обсуждении конкретной задачи.

**План релизов**. Версия / Дата / Фичи / Owner / Статус. Каждая\
строка — релиз. Обновляется командой в реальном времени.

**Env variables / API endpoints**. Имя / Описание / Пример значения.\
Стандартный способ документировать конфиги.

**Импорт данных из спредшита**. Выделили в Google Sheets диапазон →\
Cmd+C → Cmd+V в Nextdocs — получили таблицу с данными. Для\
одноразовых снимков дашбордов.

## Ограничения

* **Максимум \~50 колонок × 500 строк** в одной таблице. Выше\
  редактор начинает подтормаживать (особенно в fullscreen).

* **Нет формул и вычислений**. Для этого — Google Sheets / Excel.

* **Нет автозаполнения** (как drag-fill в Excel).

* **Нет pivot tables** (группировка, агрегация).

* **Перенос таблицы на новую страницу при экспорте PDF** — пока в\
  разработке; очень большие таблицы экспортируются с горизонтальным\
  скроллом.
